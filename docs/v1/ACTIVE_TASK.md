# Active v1 task

## T26 — Release readiness audit

Status: complete
Approved: 2026-07-20

### Scope

Audit the prepared `1.0.0-next.0` candidate against every public package,
browser, accessibility, SSR, documentation, distribution, compatibility, and
rollback requirement. Produce a reproducible go/no-go record without
publishing, tagging, redirecting stable exports, or changing private consumers.

- Verify all 32 advertised components remain conformant and aligned across the
  inventory, public pages, examples, tests, conformance records, and exports.
- Exercise a production-built v1 surface in a real browser for representative
  rendering, keyboard/form interaction, responsive navigation, overlays,
  reduced-motion, and console-error smoke coverage. Do not use screenshot loops
  as a substitute for semantic checks.
- Re-run automated accessibility, SSR/hydration, supported-browser, CSS/token,
  package, legacy-contract, bundle, and packed-consumer gates.
- Inspect the actual packed manifest and contents, confirm the stable root is
  still legacy, and prove the documented package-version rollback path.
- Record remaining risks and a clear release recommendation based only on
  public repository evidence, preserving ADR 0025.

### Expected files

- `docs/v1/RELEASE_READINESS.md` for deterministic audit evidence and the final
  recommendation.
- Release-audit or package-manifest validation under `scripts/`, plus package
  scripts where needed to make the checks repeatable.
- Existing public fixtures/tests/docs only when the audit exposes a verification
  gap; generated artifacts must be regenerated by their documented command.
- `docs/v1/ACTIVE_TASK.md`, and an ADR only if the audit establishes a new
  cross-cutting release rule.
- No legacy or v1 component runtime changes are expected. A product bug would
  require a separately approved component repair rather than being hidden in
  this audit.

### Acceptance checks

- The candidate passes `npm run verify:v1` plus a release-specific gate that
  validates package version, exports, packed contents, documentation alignment,
  legacy isolation, rollback metadata, and the absence of publish side effects.
- Real-browser smoke evidence covers representative components and interaction
  classes with no uncaught page/console errors or semantic failures.
- Accessibility, SSR/hydration, browser support, CSS/token integrity, bundle
  budgets, and Vite/Next packed-package builds are green with exact commands and
  results recorded.
- The current tree and release report contain no private-consumer information;
  no runtime source changed during T26.
- `RELEASE_READINESS.md` gives an evidence-backed go/no-go recommendation and
  explicitly leaves registry publication, remote release creation, and stable
  root cutover to later owner approval.
